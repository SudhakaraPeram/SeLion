Building SeLion from Source Code

Prerequisites

Make sure the following software is installed on your machine before trying to build SeLion:

  • Java JDK 1.7
  • Maven 3.2.x
  • Chrome (if you plan to run the build tests)
  • Chromedriver (if you plan to run the build tests)

Cloning the Repository

To build SeLion you first need to clone the code from the git repository:


  
The entire Maven project will be cloned for you. The project is
    organized as a multi-module maven project. The folder will contain a POM file as
    well as separate folders for each of the sub modules. The good thing is
    that the sub modules
    can be run and built as independent Maven projects themselves.

Project Organization

  
All the Maven projects are organized as follows:

  
    
    
  • src/main/java - Contains the code and logic of the application.
    • 
    
  • src/test/java - Contains the unit and functional tests used to check the application.
    • 
    
  • src/main/resources - Contains the non-code files required
      for the application to work.
    • 
    
  • src/test/resources - Contains all the files that are required to run the unit and functional tests. Also contains the suite
